## 8. Tech decisions & gotchas

### Why Python requests instead of wrangler
`/sessions` disk was at 100% full during development. `npm install` failed. `pip install` of some packages failed. We pivoted to native tools already available:
- `curl` or `python3 requests` for Cloudflare API
- `ImageMagick convert` for HEIC → JPG (pillow_heif install failed, ffmpeg failed on HEIC moov atom)

### How to convert HEIC files for viewing
```bash
for f in /sessions/dazzling-friendly-clarke/mnt/Maverick/Branding/*.heic; do
  base=$(basename "$f" .heic)
  convert "$f" -resize 900x "/tmp/yanik-doodles/view/${base,,}.jpg"
done
```

Then copy to `/sessions/dazzling-friendly-clarke/mnt/outputs/yanik-doodles-view/` so the Read tool (which uses the Mac host path) can access them.

### Path mapping recap
- File tool / Read / Write / Edit → uses Mac host paths under `/Users/jamesguldan/...`
- Bash workspace → uses `/sessions/dazzling-friendly-clarke/mnt/` paths
- The photos folder via bash: `/sessions/affectionate-tender-fermi/mnt/Maverick/Yanik Personal Brand Site Photos/`
- The photos folder via file tools: `/Users/jamesguldan/Library/CloudStorage/GoogleDrive-james@jamesguldan.com/My Drive/02 Client Projects/Maverick/Yanik Personal Brand Site Photos/`

### Worker bundle format
Current worker uses basic ESM:
```js
export default {
  async fetch(request, env, ctx) {
    return new Response(HTML, { headers: { "content-type": "text/html; charset=utf-8" }});
  }
}
```

HTML is wrapped in a template literal with backticks and ${ escaped. Use the `/tmp/build-worker3.py` pattern (or equivalent) to rebuild from an updated HTML source.
