# Design Handoff: Yanik Silver Personal Brand Site

## What this is

The full working package for any design work on Yanik Silver's personal brand site. Primed for the current myth storybook polish pass, and now also set up so you can take on any of the remaining deck sections without needing another brief.

## Read these in order

1. [01-CONTEXT-BRIEF.md](./01-CONTEXT-BRIEF.md). Who Yanik is, the mythology, voice rules, color system, full 10 section deck architecture.
2. [02-POLISH-BRIEF.md](./02-POLISH-BRIEF.md). The current myth storybook polish work order. Six targeted design changes, four copy tweaks, done criteria.
3. [03-myth-storybook.html](./03-myth-storybook.html). The file to polish. Image references use absolute raw URLs pointing at the images folder in this repo so the HTML renders anywhere.

## When you are ready for the next sections

4. [NEXT-SECTIONS-RECOMMENDATIONS.md](./NEXT-SECTIONS-RECOMMENDATIONS.md). A section by section map of what to design next. Each of the seven remaining sections has its myth beat, the assets to use, the source quotes to pull from, and open design questions. Priority order is inside.

## Asset library

5. [ASSETS-README.md](./ASSETS-README.md). Every image in the `images/` folder, categorized and described. Cebu totem, 18 journal doodles, 4 tattoos, 10 Yanik portraits and composites, 10 celebrity photos, 12 testimonial avatars.

## Source material (primary sources)

6. [source-material/SOURCE-README.md](./source-material/SOURCE-README.md). Index to 13 primary source files. Yanik's 60 minute transcript, 92 named testimonials, the full project handoff, implementation plan, asset inventory, original brief, architecture plan, and doodle capture plan. Use this whenever you need to ground a copy or design decision in what Yanik actually said or what the project actually requires.

## Live deployment

Current state of the site lives at https://yaniksilver-com.pages.dev for reference.

## What to return

The polished HTML file, a per change confidence score for each design change, and any judgment calls you made. If you are working on a next section rather than the polish pass, return the new HTML, confidence scores on each design decision, and a one line rationale for each asset you pulled.

## Hard rules (non negotiable)

1. Never use dashes in copy. Not em dash, not en dash, not hyphen as pause. Commas, periods, or line breaks only.
2. Never use emojis.
3. Do not invent new art. Reuse only the images in the `images/` folder.
4. Do not AI regenerate or restyle Yanik's doodles. His hand is his hand.
5. Do not redesign the hero (Section 00). Only the dawn transition band from the polish brief.
6. Include a confidence score on every meaningful design decision.

## Cosmology note

The implementation plan (in `source-material/YANIK-IMPLEMENTATION-PLAN.md`) specifies the Cosmic Sloth origin as the Bribri tribe of Costa Rica, not Cebu. The current myth storybook uses Cebu. If you are working in the polish pass, hold the Cebu framing already baked into `03-myth-storybook.html`. If you are designing a new section, flag the discrepancy back to James before shipping copy that names the tribe, and default to the Bribri framing per the implementation plan.
